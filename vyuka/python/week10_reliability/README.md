# Week 10 – Reliability: Validation, Error Handling, Logging & Tests

Companion files for the lecture **"Reliability in Industrial Apps"**.

---

## Files

| File | Purpose |
|---|---|
| `validation.py` | Core module: `ValidationResult`, `Sample`, all validators (levels 1–5), custom exceptions |
| `test_validation.py` | pytest unit tests – one test class per validation level |
| `examples_walkthrough.py` | Step-by-step classroom walkthrough, mirrors the lecture slides |
| `generate_data.py` | Synthetic data generator – writes clean and faulty CSV files |
| `data/sensor_data_clean.csv` | Clean sensor readings (no errors) |
| `data/sensor_data_dirty.csv` | Sensor readings with hand-crafted errors (for exercises) |
| `data/sensor_data_faulty.csv` | Sensor readings with programmatically injected faults (generated by `generate_data.py`) |
| `data/events_dirty.json` | Event log with missing fields, wrong types and case issues |

---

## Quick Start

```bash
# 1. Install dependencies
pip install pytest

# 2. Run tests
pytest test_validation.py -v

# 3. Run the walkthrough
python examples_walkthrough.py

# 4. Regenerate synthetic data
python generate_data.py
```

## Exercises

1. **Level 1:** Open `sensor_data_dirty.csv` and write a function that detects all rows where `value` is not a valid float. Count them.

2. **Level 2:** Add a new signal `rpm` with range `[0, 3600]` RPM to `validation.py` and write a pytest test for it.

3. **Level 3:** Find the row in `sensor_data_dirty.csv` where the rate-of-change of `vibration_rms` exceeds 2 mm/s².

4. **Level 4:** Parse `events_dirty.json` and find all events where `ts` is `null` or unparseable.

5. **Level 5:** Write a `validate_motor_state(rpm, current_a)` function: if RPM > 0 but current < 0.5 A, flag `consistency:no_load_current`.

6. **Logging:** Add a `RotatingFileHandler` to `examples_walkthrough.py` that writes `DEBUG`-level messages to `debug.log` while showing only `WARNING` and above on the console.

7. **Testing:** Add a parametrized pytest test (`@pytest.mark.parametrize`) for `validate_range` that covers at least 5 different out-of-range values for the temperature signal.
